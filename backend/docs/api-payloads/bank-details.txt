ENDPOINT
Route: /api/policy/bank-details
Method: POST
Authentication Required: Yes
Roles Allowed: Any authenticated agent
Description: Adds or updates bank details on an existing policy.

Headers:
- Authorization: Bearer <token>
- Content-Type: application/json

Request Body Example:
{
  "policy_id": "policy-id",
  "bank_name": "Nepal Bank Limited",
  "bank_account": "1234567890",
  "branch": "Kathmandu Main Branch",
  "premium_due_date": "2026-12-31",
  "premium_paid": 5000
}

Validation Rules:
- policy_id: required
- bank_account: required
- branch: required
- premium_due_date: required
- premium_paid: required, must be a positive number or zero
- bank_name: optional but must be non-empty if supplied

Success Response Example:
{
  "status": true,
  "message": "Bank details added successfully",
  "data": {
    "policy": {
      "id": "policy-id",
      "bank_name": "Nepal Bank Limited",
      "bank_account": "1234567890",
      "branch": "Kathmandu Main Branch",
      "premium_due_date": "2026-12-31",
      "premium_paid": 5000
    }
  },
  "code": 200
}

Possible Error Responses:
- 400 validation failed
- 401 missing token
- 404 policy not found
- 500 failed to add bank details

MongoDB Collections Affected:
- policy

Business Logic Notes:
- The policy is fetched by id and then updated in place.
- This endpoint is the dedicated bank-details update flow used after enrollment or policy creation.
