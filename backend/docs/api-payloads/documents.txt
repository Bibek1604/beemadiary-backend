==================================================
ENDPOINT
Route: /api/client/documents
Method: POST
Authentication Required: Yes
Roles Allowed: Any authenticated agent
Description: Uploads a client profile picture plus supporting documents and photos, then persists the uploaded metadata on the client record.

Headers:
- Authorization: Bearer <token>
- Content-Type: multipart/form-data

Request Body Example:
{
  "client_id": "client-id"
}

Multipart File Fields:
- profile_picture: single image file, maxCount 1
- supporting_documents: array of files, maxCount 5
- photos: array of image files, maxCount 10

Validation Rules:
- client_id is required
- client must belong to the authenticated agent
- file types allowed: image/jpeg, image/png, image/webp, image/gif, application/pdf, application/msword, application/vnd.openxmlformats-officedocument.wordprocessingml.document

Success Response Example:
{
  "status": true,
  "message": "Documents uploaded successfully",
  "data": {
    "client_id": "client-id",
    "uploads": {
      "profile_picture": {
        "url": "https://...",
        "public_id": "client-id-profile",
        "filename": "profile.jpg"
      },
      "supporting_documents": [
        {
          "url": "https://...",
          "public_id": "client-id-doc-1",
          "filename": "id-card.pdf"
        }
      ],
      "photos": [
        {
          "url": "https://...",
          "public_id": "client-id-photo-1",
          "filename": "front.jpg"
        }
      ]
    }
  },
  "code": 200
}

Possible Error Responses:
- 400 client_id missing
- 401 missing token
- 404 client not found or not owned by the agent
- 500 failed to upload documents

MongoDB Collections Affected:
- client

Business Logic Notes:
- profile_picture is stored as a single field on the client record.
- supporting_documents are appended to the documents array.
- photos are appended to the images array.
- This route is distinct from enrollment image handling.
