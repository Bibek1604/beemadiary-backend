ENDPOINT
Route: /api/auth/register
Method: POST
Authentication Required: No
Roles Allowed: Public
Description: Register a new user record in the shared user collection. The TypeScript service defaults the role to AGENT.

Headers:
- Content-Type: application/json

Request Body Example:
{
  "email": "new.agent@example.com",
  "password": "ChangeMe123!",
  "first_name": "New",
  "last_name": "Agent"
}

Validation Rules:
- email: required, valid email
- password: required, minimum 12 characters
- first_name: required, non-empty string
- last_name: required, non-empty string

Success Response Example:
{
  "status": true,
  "message": "User registered successfully",
  "data": {
    "user": {
      "id": "uuid",
      "email": "new.agent@example.com",
      "firstName": "New",
      "lastName": "Agent",
      "role": "AGENT"
    }
  },
  "code": 201
}

Possible Error Responses:
- 400 validation error or weak password
- 409 user already exists
- 500 unexpected failure

MongoDB Collections Affected:
- user

Business Logic Notes:
- Always creates AGENT users unless another role is explicitly used by the service layer.
- Passwords are hashed before persistence.

ENDPOINT
Route: /api/auth/login
Method: POST
Authentication Required: No
Roles Allowed: Public
Description: Generic login that returns access token, refresh token, and active session metadata.

Headers:
- Content-Type: application/json

Request Body Example:
{
  "email": "agent@test.com",
  "password": "Agent@123456"
}

Validation Rules:
- email: required, valid email
- password: required, non-empty string
- optional device_name may be accepted by controller layer

Success Response Example:
{
  "status": true,
  "message": "Login successful",
  "data": {
    "user": {
      "id": "uuid",
      "email": "agent@test.com",
      "firstName": "Test",
      "lastName": "Agent",
      "role": "AGENT"
    },
    "tokens": {
      "accessToken": "jwt-access-token",
      "refreshToken": "jwt-refresh-token"
    },
    "session": {
      "id": "session-id",
      "expiresAt": "2026-05-30T12:00:00.000Z"
    }
  },
  "code": 200
}

Possible Error Responses:
- 400 invalid credentials or validation error
- 403 inactive account
- 500 unexpected failure

MongoDB Collections Affected:
- user
- session
- refreshToken
- auditLog

Business Logic Notes:
- Creates a new session record and refresh token on successful login.
- The token payload includes id, email, role, and type.

ENDPOINT
Route: /api/auth/admin/login
Method: POST
Authentication Required: No
Roles Allowed: Public
Description: Legacy/admin login flow that authenticates against the admin collection and returns a single JWT access token.

Headers:
- Content-Type: application/json

Request Body Example:
{
  "email": "admin@beemadiary.com",
  "password": "kathmandu@kts@123"
}

Validation Rules:
- email: required, valid email
- password: required, non-empty string

Success Response Example:
{
  "status": true,
  "message": "Login successful",
  "token": "jwt-access-token",
  "data": {
    "id": "uuid",
    "email": "admin@beemadiary.com"
  }
}

Possible Error Responses:
- 400 invalid credentials
- 403 account deactivated
- 429 too many login attempts if the route layer adds rate limiting
- 500 unexpected failure

MongoDB Collections Affected:
- admin
- session
- auditLog

Business Logic Notes:
- Uses the CommonJS auth controller and returns the compact admin payload.
- The route body is documented in Swagger and should use AdminLoginRequest.

ENDPOINT
Route: /api/auth/logout
Method: POST
Authentication Required: Yes
Roles Allowed: Any authenticated user
Description: Terminates the current session and clears cookies.

Headers:
- Authorization: Bearer <token>
- Content-Type: application/json

Request Body Example:
{
  "session_id": "session-id"
}

Validation Rules:
- auth token required
- session_id is used by the service layer when present

Success Response Example:
{
  "status": true,
  "message": "Logout successful",
  "code": 200
}

Possible Error Responses:
- 401 unauthenticated request
- 500 unexpected failure

MongoDB Collections Affected:
- session

Business Logic Notes:
- Clears accessToken and refreshToken cookies.

ENDPOINT
Route: /api/auth/refresh
Method: POST
Authentication Required: No, but refreshToken is required in body or cookie
Roles Allowed: Public
Description: Rotates refresh token and issues a new access token pair.

Headers:
- Content-Type: application/json

Request Body Example:
{
  "refreshToken": "jwt-refresh-token"
}

Validation Rules:
- refreshToken is optional in schema but required in practice if cookie is absent

Success Response Example:
{
  "status": true,
  "message": "Token refreshed",
  "data": {
    "tokens": {
      "accessToken": "new-access-token",
      "refreshToken": "new-refresh-token"
    }
  },
  "code": 200
}

Possible Error Responses:
- 401 missing or invalid refresh token
- 500 unexpected failure

MongoDB Collections Affected:
- refreshToken
- session

Business Logic Notes:
- Old refresh token is revoked before the new one is stored.

- admin
- agent
- client

- Password strength validation is enforced by the service layer.
