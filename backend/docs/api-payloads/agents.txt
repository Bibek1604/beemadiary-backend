ENDPOINT
Route: /api/agent/profile
Method: GET
Authentication Required: Yes
Roles Allowed: AGENT
Description: Returns the current agent profile from the agent collection.

Headers:
- Authorization: Bearer <token>

Request Body Example:
{}

Success Response Example:
{
  "status": true,
  "message": "Agent profile retrieved successfully",
  "data": {
    "id": "uuid",
    "full_name": "Test Agent",
    "email": "agent@test.com",
    "phone_number": "9999999999",
    "status": "ACTIVE",
    "company_id": "company-id"
  },
  "code": 200
}

Possible Error Responses:
- 401 missing token
- 403 not an agent
- 404 profile not found
- 500 unexpected failure

MongoDB Collections Affected:
- agent

Business Logic Notes:
- Null and empty values are stripped from the response before returning.

ENDPOINT
Route: /api/agent/profile
Method: POST
Authentication Required: Yes
Roles Allowed: AGENT
Description: Create or update the current agent profile and optionally upload a profile image.

Headers:
- Authorization: Bearer <token>
- Content-Type: multipart/form-data

Request Body Example:
{
  "full_name": "Test Agent",
  "phone_number": "9999999999",
  "license_number": "LIC-123",
  "license_expiry": "2027-12-31",
  "branch": "Kathmandu",
  "designation": "Senior Agent",
  "qualification": "Bachelor",
  "bio": "Experienced insurance agent",
  "years_of_experience": 5,
  "specialization": "Life insurance",
  "profile_image_url": "https://..."
}

Multipart File Fields:
- image: single image file

Validation Rules:
- full_name: required, non-empty string
- phone_number: optional, valid phone format if supplied
- license_number: optional, minimum 3 characters if supplied
- image: optional image file, saved to Cloudinary when provided

Success Response Example:
{
  "status": true,
  "message": "Agent profile updated successfully",
  "data": {
    "id": "uuid",
    "full_name": "Test Agent",
    "phone_number": "9999999999",
    "lic_agent_code": "LIC-123",
    "profile_picture": "https://...",
    "profile_picture_public_id": "agent-profile-uuid"
  },
  "code": 200
}

Possible Error Responses:
- 400 validation error or image upload failure
- 401 missing token
- 500 unexpected failure

MongoDB Collections Affected:
- agent

Business Logic Notes:
- The uploaded image is stored as the single agent profile picture, not as an array.

ENDPOINT
Route: /api/agent/profile/upload-image
Method: POST
Authentication Required: Yes
Roles Allowed: AGENT
Description: Uploads only the agent profile image and updates the agent record with the Cloudinary URL and public id.

Headers:
- Authorization: Bearer <token>
- Content-Type: multipart/form-data

Request Body Example:
{}

Multipart File Fields:
- image: single image file

Validation Rules:
- image required
- file types allowed: JPEG, PNG, WebP, GIF
- max size: 5MB

Success Response Example:
{
  "status": true,
  "message": "Image uploaded successfully",
  "data": {
    "image_url": "https://...",
    "public_id": "agent-profile-uuid",
    "file_size": 123456,
    "file_type": "image/png",
    "upload_provider": "Cloudinary"
  },
  "code": 200
}

Possible Error Responses:
- 400 invalid file or missing image
- 401 missing token
- 500 upload failure

MongoDB Collections Affected:
- agent

Business Logic Notes:
- This is the single-image path for agent profile photos.

ENDPOINT
Route: /api/agent/notifications
Method: GET
Authentication Required: Yes
Roles Allowed: AGENT
Description: Returns notifications targeted to the current agent, with pagination and read-status filtering.

Headers:
- Authorization: Bearer <token>

Query Parameters:
- page: optional integer, default 1
- limit: optional integer, default 10
- search: optional string
- is_read: optional boolean

Request Body Example:
{}

Success Response Example:
{
  "status": true,
  "message": "Notifications fetched successfully",
  "data": {
    "notifications": [
      {
        "id": "uuid",
        "title": "Premium Reminder",
        "content": "Submit pending premium today.",
        "target_type": "SINGLE",
        "created_at": "2026-05-30T10:00:00.000Z",
        "creator": {
          "id": "admin-id",
          "username": "admin",
          "email": "admin@beemadiary.com"
        },
        "is_read": false,
        "read_at": null
      }
    ],
    "pagination": {
      "total": 1,
      "page": 1,
      "limit": 10,
      "totalPages": 1
    }
  },
  "code": 200
}

Possible Error Responses:
- 400 query validation error
- 401 missing token
- 403 not an agent
- 500 unexpected failure

MongoDB Collections Affected:
- bulkNotification
- notificationRead
- admin
- agent

Business Logic Notes:
- The service combines ALL notifications and SINGLE notifications addressed to the current agent.
