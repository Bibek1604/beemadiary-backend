Business Logic Notes:
- The backend uses a MongoDB adapter that maps Prisma-style calls onto collections directly.
- Authentication routes return different payload shapes depending on whether they use the TypeScript auth service or the legacy JS admin/agent flow.
- The active enrollment flow stores one profile picture in profile_picture and supporting images as an array in images.
- Client document uploads are separate from enrollment images and store supporting_documents and photos independently.
- If a route is documented in both TS and JS files, the master reference keeps the active app route plus the legacy alias.
